# お得情報自動収集ボット

**ファイル構成をシンプルにしています。今後普段いじるのは`config.yaml`だけ、
コード自体を直したい時だけ`main.py`(1ファイル)を編集すれば済むようにしてあります。**

```
config.yaml          ← 情報源・閾値・通知先の設定(普段はここだけ編集)
main.py              ← 全ロジックが1ファイルにまとまっている(たまに編集)
requirements.txt      ← 依存パッケージ一覧(通常は編集不要)
.github/workflows/scrape.yml  ← 自動実行の設定(通常は編集不要)
docs/index.html       ← 閲覧用ページ(通常は編集不要)
docs/deals.json        ← 自動生成される一覧データ(手で編集しない)
data/seen_items.json   ← 自動生成される重複防止データ(手で編集しない)
```

## 初回セットアップ(PCで)

1. GitHubに新しいリポジトリを作成
2. このフォルダの中身を全部リポジトリにpush(`git init` → `git add .` → `git commit` → `git push`、
   またはGitHub Desktop等でもOK)
3. Discordで通知先チャンネルのWebhook URLを発行
   (チャンネル設定→連携サービス→ウェブフック→新しいウェブフック)
4. GitHubリポジトリの Settings → Secrets and variables → Actions →
   `DISCORD_WEBHOOK_URL` という名前でWebhook URLを登録
5. Settings → Pages → Source: Deploy from a branch、Branch: main、フォルダ: /docs → Save
   → 数分後に `https://ユーザー名.github.io/リポジトリ名/` が閲覧用URLになる
6. Actionsタブ → ワークフローを選択 → Run workflow で手動実行して動作確認

### 見た目(HTML)を直したい
`docs/index.html` を開いて鉛筆アイコンで編集。CSS・JS・HTML全部が1ファイルに
入っていて、ビルド作業は一切ないので保存すればそのまま反映されます。
ファイル内は見出しコメントで区切ってあるので、直したい場所を検索してください。

| 直したいこと | 探すコメント |
|---|---|
| 色を変えたい | `===== 色・フォントの設定 =====` |
| タイトル部分の見た目 | `===== ヘッダー =====` |
| カテゴリの絞り込みタグの見た目 | `===== カテゴリフィルタのチップ =====` |
| カード1件分のデザイン | `===== 左側の値札風タグ =====` `===== カード内のテキスト部分 =====` |
| カードに表示する情報や並び替えロジック | `===== カード1件分のHTML組み立て =====`(JS) |
| 「情報がありません」の文言 | `===== 空の状態 =====` |

## 今後の運用(スマホだけでOK)

### 新しいセール情報源を追加したい
GitHubアプリ or ブラウザで `config.yaml` を開く→ 鉛筆アイコンで編集 →
`sources` に同じ形式でブロックを追記 → 画面下部から直接コミット。

```yaml
  - name: "追加したいサイト名"
    type: "news_watch"           # 基本はこれが一番安定(Googleニュース見出し監視)
    query: "検索したいキーワード"
    category: "家電・通販"        # 好きな分類名でOK
```

### 閾値やキーワードを調整したい
`config.yaml` の `filter` セクション(`min_discount_percent`など)を編集するだけ。

### コード自体の動きを変えたい
`main.py` を開いて該当セクション(ファイル内の`# ===== ○○ =====`という
見出しコメント)を編集。GitHubアプリ・ブラウザどちらでも鉛筆アイコンから
直接編集してコミットできます。

## カテゴリ分類

- 🛒 家電・通販: 価格.com、ビックカメラ、ヨドバシ、ヤマダ電機、ノジマ
- 🏠 生活雑貨・家具: ニトリ、無印良品週間
- 💰 ポイント還元: PayPayキャンペーン等
- 🔥 セール速報: ゾゾ祭り、楽天スーパーSALE、Amazonプライムセール、Qoo10メガ割
- ✈️ 旅行: ANA/JALセール、LCC、楽天トラベル/じゃらん
- 🎫 イベント・レジャー: テーマパーク割引、映画・美術館の割引デー

## 情報源のタイプ一覧(config.yamlの`type`)

| type | 説明 | 安定性 |
|---|---|---|
| `news_watch` | Googleニュースの見出しを監視。新しいサイトを追加するならこれが一番簡単で壊れにくい | ◎ |
| `rss_watch` | サイトが公式に配信しているRSS/Atomフィードを直接取得 | ◎ |
| `point_campaign` | ページ内のテキストから「○%OFF」「○%還元」を正規表現で抽出 | ○ |
| `kakaku_pricedown` | 価格.comの値下がりランキングページ専用 | ○ |
| `generic_html` | CSSセレクタを自分で指定してスクレイピング。サイト構造が変わると壊れやすい | △ |

新しく追加する情報源で迷ったら、まず`news_watch`を試すのがおすすめです。

## 現状の制約
- スクレイピング系(`generic_html`, `kakaku_pricedown`)はサイトのHTML構造が
  変わると動かなくなることがあります。「フィルタ後の該当件数」が急に0件に
  なったら要確認です(Actionsタブの実行ログで確認できます)。
- GitHub Pagesを無料で使うにはリポジトリを**Public**にする必要があります。
- LINE Notifyは2025年3月末で終了済みのため、LINE通知を使う場合は
  LINE Messaging APIのpush API経由になります(`config.yaml`の`notify.line`)。
